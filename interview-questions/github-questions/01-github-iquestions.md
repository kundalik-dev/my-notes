🔥 Most Common Git Interview Questions for QA
1️⃣ What is Git and why do we use it in Automation?2️⃣ Difference between Git and GitHub.3️⃣ Explain Git workflow (Working Directory → Staging Area → Repository).4️⃣ How do you check the status of files in Git?5️⃣ Difference between “fetch” and “pull”.6️⃣ How do you undo changes in Git? (reset, revert, checkout)7️⃣ How do you resolve merge conflicts?8️⃣ Difference between “merge” and “rebase”.9️⃣ How do you clone a repository?🔟 How do you create and switch branches?

⏳ Learn Git in 5 Minutes – QA Version
💡 Just remember these 7 steps:

1. Clone the repository to get it locally.
2. Check the status of files to see changes.
3. Stage the files you want to commit.
4. Commit with a meaningful message.
5. Push changes to the remote branch.
6. Pull the latest updates from the remote branch.
7. Switch/Create branches for new features or fixes.
