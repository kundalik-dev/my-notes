## Writing Guidelines

- Write in simple, clear English.
- Keep notes practical and interview-focused.
- Prefer short paragraphs and bullet points.
- Add examples from QA projects.
- Add code only when it helps revision.
- Explain concepts step by step for new topics.
- For communication practice, include a natural spoken answer.
- Avoid unnecessary theory unless it is commonly asked in interviews.

## Code Guidelines

For C# Selenium/NUnit:

- Use Selenium 4 style.
- Prefer explicit waits.
- Avoid `Thread.Sleep` unless explaining why not to use it.
- Use Page Object Model examples.
- Use NUnit attributes: `[SetUp]`, `[Test]`, `[TearDown]`.

For Java Selenium/TestNG:

- Keep examples simple for revision.
- Explain annotations like `@BeforeMethod`, `@Test`, and `@AfterMethod`.
- Compare with NUnit when useful.

For Playwright:

- Prefer JavaScript/TypeScript examples using `@playwright/test`.
- Use good locators such as `getByRole`, `getByLabel`, `getByText`, and `getByTestId`.
- Show assertions with `expect`.
- Mention Playwright auto-waiting.

For API testing:

- Start from basics: HTTP methods, status codes, headers, body, authentication, Postman.
- Then move to automation examples.
- Explain validation clearly: status code, response body, headers, schema, response time.
